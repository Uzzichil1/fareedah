# TinyKloset

A mobile-first, installable (PWA) peer-to-peer marketplace for **curated pre-loved
and boutique baby & children's clothing, footwear, and accessories**.

> **Status: Phase 2 (data model + auth).** The data model and authentication
> (email/password + Google, role-based access) are in place. Listings, curation,
> checkout, payments, shipping, and messaging are built in later phases.

## Stack

| Concern        | Choice |
| -------------- | ------ |
| Framework      | Next.js 16 (App Router) + TypeScript (strict) |
| Styling        | Tailwind CSS v4 (CSS-first config) |
| Database       | PostgreSQL (Supabase) via Prisma 7 (driver adapters) |
| Auth           | Auth.js (NextAuth) — email/password + Google + Apple *(Phase 2)* |
| Payments       | Stripe Connect — escrow + commission *(Phase 4)* |
| Shipping       | Shippo prepaid labels *(Phase 4)* |
| Images         | Cloudinary upload/resize/CDN *(Phase 3)* |
| Validation     | Zod + React Hook Form |
| PWA            | Native manifest (`app/manifest.ts`) + service worker (`public/sw.js`) |

## Prerequisites

- Node.js 20+ (developed on Node 24)
- npm
- A Supabase project (free tier is fine) for the Postgres database — needed from
  Phase 2 onward. Phase 1's home page runs without a live database.

## Setup

```bash
# 1. Install dependencies (also runs `prisma generate` via postinstall)
npm install

# 2. Create your local env file and fill in values
cp .env.example .env        # PowerShell: Copy-Item .env.example .env

# 3. Generate the Prisma client (also runs automatically on install)
npm run db:generate

# 4. Start the dev server
npm run dev
# open http://localhost:3000
```

### Environment variables

Every variable the app reads is documented in [`.env.example`](./.env.example).
Phase 1 only needs `DATABASE_URL`/`DIRECT_URL` placeholders and `AUTH_SECRET`;
the remaining keys (Stripe, Shippo, Cloudinary, Google/Apple) are wired in later
phases. **Never commit `.env`.**

#### Supabase connection strings

From the Supabase dashboard → *Project Settings → Database → Connection string*:

- `DATABASE_URL` — the **Transaction pooler** URL (host `...pooler.supabase.com`,
  port **6543**). Used by the app at runtime through Prisma's pg driver adapter.
- `DIRECT_URL` — the **direct/session** URL (port **5432**). Used by
  `prisma migrate` (configured in `prisma.config.ts`).

## Database (Prisma 7)

Prisma 7 is Rust-free and uses **driver adapters** plus a `prisma.config.ts`
file; connection URLs no longer live in `schema.prisma`.

- `prisma/schema.prisma` — models + the `prisma-client` generator (outputs to
  `src/generated/prisma`, which is gitignored and regenerated on install).
- `src/lib/db.ts` — the singleton `PrismaClient` wired to the `@prisma/adapter-pg`
  driver using the pooled `DATABASE_URL`.
- `prisma.config.ts` — migration config using the direct `DIRECT_URL`.

```bash
npm run db:generate   # regenerate the client after schema changes
npm run db:migrate    # create/apply a migration (needs a live DIRECT_URL — Phase 2)
```

## PWA

- `src/app/manifest.ts` is served at `/manifest.webmanifest`.
- `public/sw.js` is a service worker with a real fetch handler (network-first
  navigations, cache-first static assets, `/offline` fallback). It is registered
  **only in production** (`src/components/ServiceWorkerRegistrar.tsx`) so it
  never caches stale assets during local development.
- Icons live in `public/icons/` and are generated by `npm run icons`
  (placeholder art — replace with final branding later).

## Project structure

```

  schema.prisma            # data model + generator
prisma.config.ts           # Prisma 7 migration/datasource config
public/
  icons/                   # PWA icons (192, 512, apple-touch)
  sw.js                    # service worker
scripts/
  generate-icons.mjs       # placeholder icon generator
src/
  app/                     # App Router routes
    layout.tsx             # root layout + PWA metadata
    page.tsx               # home page
    manifest.ts            # web app manifest
    offline/page.tsx       # offline fallback
  components/              # shared UI / client components
  generated/prisma/        # generated Prisma client (gitignored)
  lib/
    db.ts                  # Prisma client singleton
.env.example               # every env var, documented
```
